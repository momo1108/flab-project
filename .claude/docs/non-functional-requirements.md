# 비기능 요구사항

## 성능 (Performance)

### 로드 시간

- **NF-1.1**: 페이지 초기 로드 시간 < 3초
- **NF-1.2**: 이미지 로딩 최적화 (TMDB 이미지 크기 제한)
  - 카드용: `w500`
  - 배경용: `original`
  - 프로필용: `w185` 또는 `h632`
- **NF-1.3**: API 응답 캐싱 (Tanstack Query)
  - 메인 피드(인기 영화/장르/테마/컬렉션): 5분 staleTime
  - 아티스트 출연작: 10분 staleTime
  - 상세페이지: 10분 staleTime
  - 검색결과: 3분 staleTime

### 렌더링 최적화

- **코드 분할**: 페이지별 라우트 기반 분할
- **Lazy Loading**: Intersection Observer 적용
- **Debouncing**: 검색 입력 300ms 디바운싱

## 사용성 (Usability)

### 반응형 디자인

- **NF-2.1**: 반응형 디자인 (모바일/태블릿/데스크톱)
- 뷰포트 너비에 따른 동시 노출 개수 범위 준수
  - 랜덤 컬렉션 Landscape: 1~3개
  - 랜덤 장르/테마/아티스트 Portrait: 1~8개
  - TOP 20 Portrait: 1~7개

### 상호작용

- **NF-2.2**: 마우스 호버 효과 (카드)
  - 확대 + 그림자 효과
  - 트랜지션 애니메이션
- **NF-2.3**: 모바일 터치 친화적 UI
  - 스와이프 제스처 지원
  - 적절한 버튼 크기 (최소 44x44px)

---

## 신뢰성 (Reliability)

### 에러 처리

- **NF-3.1**: API 에러 재시도 (최대 1회)
- **NF-3.2**: 네트워크 오류 시 사용자 친화적 메시지
  - 섹션 단위 fallback UI
  - 재시도 버튼 제공

### 데이터 무결성

- API 응답 검증
- 이미지 로딩 실패 시 플레이스홀더
- 유효하지 않은 ID 처리 (404 상태)

---

## 보안 (Security)

### API 관리

- **NF-4.1**: API Key는 환경변수로 관리
  - `.env` 파일 사용
  - `.gitignore`에 포함
  - 프로덕션 환경에서는 안전하게 관리

### 전송 보안

- **NF-4.2**: HTTPS 전송 (CloudFront 기본)
- 민감 정보는 URL 파라미터에 포함하지 않음
- XSS 방지 (React 기본 보호)

## 확장성 (Scalability)

### 코드 구조

- **NF-5.1**: CSS Modules로 스타일 격리

- API 로직 분리

- 키보드 탐색 지원
- 스크린 리더 호환

- 버튼에 충분한 크기 (최소 44x44px)

## 성능 측정 기준

### 핵심 성능 지표

- **FCP (First Contentful Paint)**: < 1.5s
- **LCP (Largest Contentful Paint)**: < 2.5s
- **FID (First Input Delay)**: < 100ms
- **CLS (Cumulative Layout Shift)**: < 0.1

### 모니터링

- 개발자 도구 Performance 탭
- Lighthouse 점수: > 90
- Web Vitals 측정

## 에이전트 개발 시 주의사항

1. **성능**: 캐시 정책 준수, 이미지 최적화 적용
2. **사용성**: 반응형 디자인, 호버 효과 구현
3. **신뢰성**: 에러 처리, 재시도 로직 구현
4. **보안**: API Key 환경변수 관리
5. **확장성**: 컴포넌트 재사용, 모듈화
6. **접근성**: 키보드 탐색, 스크린 리더 호환
